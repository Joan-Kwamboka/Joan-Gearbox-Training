Project Management and Documentation

Through out this training we will be reporting all our progress, learning and assignments on a personal website. We were taken through the basics of how to build a website using HTML.

We installed Visual Studio Code to help us with our coding and Git to be able to track changes in our source code. We made Github accounts and learnt how to use the inbuilt Terminal function of Visual Studio Code to push and pull our changes to GitHub.

Git is a tool used to track changes in your code on your computer, while GitHub is an online platform used to store and share your code. Git helps you manage your work locally, and GitHub allows you to back it up and collaborate with others online.

Key Takeaways

  • git status
    This command shows the current state of your project. It tells you which files have been changed, which ones are ready to be saved (staged), and which ones are not yet tracked by Git.
  • git pull
    Downloads the latest changes from your GitHub repository to your computer. It ensures you are working with the most up-to-date version of your project before making new changes.
  • git add .
    Prepares all the modified files to be saved. It tells Git that you want to include these changes in your next update (commit). You can also add a specific file using git add filename.
  • git commit -m "message"
    Saves your changes locally. The message should briefly explain what changes you made, so you can easily understand your progress later.
  • git push origin main
    Uploads your saved changes (commits) to GitHub. This makes your work available online and acts as a backup of your project.
