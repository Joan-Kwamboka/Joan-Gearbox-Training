Introduction to Embedded Systems

We were introduced to the concept of embedded systems and their applications in various industries. Embedded Systems are devices built for a dedicated function often in real time unlike general purpose computers.

Core Components

  • Microcontroller/Microprocessor: The brain of the embedded system that processes data and executes instructions. A microprocessor is just a CPU that needs external memory and peripherals to function, while a microcontroller is an all-in-one chip with CPU, memory, and I/O built in for controlling devices.
  • Memory: Used to store data and program code. It can be volatile (RAM) or non-volatile (Flash).
  • Input/Output Interfaces: Allow the embedded system to interact with the external environment through sensors, actuators, and communication ports.
  • Software: The embedded code that runs on the hardware to perform specific tasks. It is often written in low-level programming languages such for efficiency.

Embedded systems are often optimized for low power consumption, compact size, and high efficiency. They are used in a wide range of applications including consumer electronics, automotive systems, medical devices, industrial automation, and more. An example application of embedded systems is electronic access control systems which use microcontrollers to manage security and access permissions.

During one of the sessions, we observed the features of the above microcontroller development boards, Arduino, ESP32, and Raspberry Pi Pico.

The main parts of the boards include;

  • Microcontroller chip: The central processing unit that executes the code and controls the board's functions.
  • GPIO Pins: General Purpose Input/Output pins used for connecting sensors, actuators, and other peripherals.
  • Power Supply: Provides the necessary voltage to operate the board, often through USB or external power sources.
  • Communication Interfaces: Such as USB, Wi-Fi, Bluetooth, or serial ports for programming and data transfer.

Assignment

For this assignment, the objective was to learn about Classes, Inheritance and Polymorphism.

I built a geometry calculator using Object-Oriented Programming (OOP) concepts in Python. I wrote and ran my code on Thonny. 

import math
class Shape:
    def area(self):
        pass

class Circle(Shape):
    def __init__(self, diameter):
        self.diameter = diameter
    def area(self):
        radius = self.diameter / 2
        return math.pi * radius ** 2
        pass

class Square(Shape):
    def __init__(self, length):
        self.length = length
    def area(self):
        return self.length ** 2 
        pass

class Rectangle(Shape):
    def __init__(self, length, width):
        self.length = length
        self.width = width
    def area(self):
        return self.length * self.width
        pass

class RightTriangle(Shape):
    def __init__(self, base, height):
        self.base = base
        self.height = height
    def area(self):
        return 0.5 * self.base * self.height
        pass

circle = Circle(10)
square = Square(5)
rectangle = Rectangle(4, 6)
triangle = RightTriangle(3, 8)
triangle2 = RightTriangle(2,10)

print("Circle area:", circle.area())
print("Square area:", square.area())
print("Rectangle area:", rectangle.area())
print("Triangle area:", triangle.area())
print("Triangle2 area:", triangle2.area())

The output when running the code was:

From this assignment I was able to understand the concept of inheritance and polyporphism.

The program demonstrated inheritance by creating a base class Shape and allowing other classes such as Circle, Square, Rectangle, and RightTriangle to inherit from it. These subclasses reuse the structure of the parent class while defining their own properties.

Polymorphism was demonstrated by how each subclass overrides the area() method of the Shape class and implements it differently depending on the shape. When the area() method is called on different objects, Python executes the appropriate method for that specific shape, allowing the same method name to calculate different areas depending on the object.
