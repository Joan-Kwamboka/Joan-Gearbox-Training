Introduction to Embedded Systems

We learnt about the compilation process where human-readable code is converted to binary instructions followed by specific hardware-level execution steps

Compilation turns the source code (like C or C++) into object code,linking then takes the object code files and any library code and connects them together so the program can run as one complete executable.

In Static Linking the necessary library code is copied directly into the final executable file during the build process the resulting file is a single, self-contained executable that includes everything it needs to run.

In contrast,in Dynamic Linking the program only includes a reference to the needed libraries during the build process. The actual linking and loading of the library code into memory happens at runtime, when you execute the program.

Steps

  • Preprocessing: Involves preparing the source code for the compiler by handling all statements starting with #.
  • Compilation: The preprocessed code is translated into assembly language, which is then converted into machine code (object files).
  • Assembly: The assembly code is converted into machine code, creating object files that contain the binary instructions.
  • Linking: The object files are combined with any necessary libraries to create a single executable file.
  • Loading: When the program is run, the operating system loads the executable into memory and starts execution.

Assignment

For this assignment, the objective was to learn about Functions, Objectives and Linking with Clang.

I built a geometry calculator C program.The program contained two files; main.c and geometry.c

The code in geometry.c was:


#include 

double circle_area(double diameter) {
    double radius = diameter / 2.0;
    return 3.141592653589793 * radius * radius;
}

double square_area(double length) {
    return length * length;
}

double rectangle_area(double length, double width) {
    return length * width;
}

double triangle_area(double base, double height) {
    return 0.5 * base * height;
}

The code in main.c was:


  #include 

double circle_area(double diameter);
double square_area(double length);
double rectangle_area(double length, double width);
double triangle_area(double base, double height);

int main() {
    printf("Circle area: %.2f\n", circle_area(10));
    printf("Square area: %.2f\n", square_area(5));
    printf("Rectangle area: %.2f\n", rectangle_area(4, 6));
    printf("Triangle area: %.2f\n", triangle_area(3, 8));
    return 0;
}

The output after running the code was:
